Headings

< h1 > , < h2 >, ..., < h6 >

These elements are used to create a hierarchy of titles and headings on a webpage. < h1 > represents the most important title, while < h6 > represents the least important.

Headers visual prominence can be achieved through variations in font size and weight. Generally, < h1 > headers are rendered with the largest font size and the boldest weight, making them stand out as the main title of the page. As you move down the hierarchy, each subsequent header level might have slightly smaller font sizes and less bold weight. This visual progression helps users quickly identify the relative importance of different sections.

Paragraphs

< p >

Paragraphs are used to group together related sentences and create distinct blocks of text. They provide a natural separation between content and enhance readability.

The use of < p > elements contributes to the semantic structure of your HTML document. It helps browsers and assistive technologies understand the organization of your content, enhancing accessibility for users, including those who rely on screen readers.

<p> Paragraph </p>

Buttons

< Input >

The primary content of a button is typically the text that appears on the button face. This text should provide a clear indication of the action the button will perform when clicked.

  • The type attribute specifies the type of button. Common values are "button" (default), "submit" (for form submission), and "reset" (to reset form fields).
  • The disabled attribute can be added to make the button unclickable and visually indicate that the action is unavailable.
  • The name attribute is often used when buttons are part of a form and are used to submit data.
  • You can use JavaScript to add custom functionality to buttons, making them perform specific tasks when clicked.

    • Proper use of the button element enhances accessibility for users who rely on assistive technologies. Screen readers, for example, interpret buttons correctly, and users can activate them using keyboard navigation.

    Unordered and Ordered Lists

    Unordered List < ul >

    An unordered list is used to create a list of items with bullet points. It is ideal for presenting information that doesn't require a specific order.

    Within the < ul > element, each list item is represented by the < li > element. This element contains the content of each list item.

    <li> Item1 </li>

    <li> Item2 </li>

    Formatted Text

    < i > , < b > , < u > , < strong >

  • < i > : Represents text in italic or emphasized style. It's typically used to convey that the text has a different emphasis or meaning.
  • < b > : Represents text in bold style. It's used to make the text visually stand out, often for keywords or important content.
  • < u > : Represents underlined text. While not recommended for general use due to its association with hyperlinks, it can be used to indicate text needing special attention.
  • < strong > : Represents strongly emphasized text. It's similar to < b > , but it carries a semantic meaning of higher importance, often used for critical content.
